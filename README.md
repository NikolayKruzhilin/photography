# Николай Кружилин — Photography Portfolio

Статичный сайт-портфолио. Деплоится через **GitHub Pages**, не требует сервера и сборки.

---

## Структура папок

```
Photography_portfolio_webpage/
├── index.html
├── photos.json          ← редактируешь сюда, чтобы добавить/убрать фото
├── .nojekyll
│
├── css/                 ← стили (не трогать)
├── js/                  ← скрипты (не трогать)
│
└── photos/
    ├── weddings/        ← фото свадеб
    ├── events/          ← фото мероприятий
    ├── lovestory/       ← фото лавстори
```

## Как добавить фотографии

### 1. Положи файл в нужную папку

| Категория     | Папка                  | Формат имени файла   | Пример              |
|---------------|------------------------|----------------------|---------------------|
| Портреты      | `photos/portraits/`    | `portrait-NN.jpg`    | `portrait-01.jpg`   |
| Мероприятия   | `photos/events/`       | `event-NN.jpg`       | `event-03.jpg`      |
| Лавстори      | `photos/lovestory/`    | `lovestory-NN.jpg`   | `lovestory-01.jpg`  |
| Свадьбы       | `photos/weddings/`     | `wedding-NN.jpg`     | `wedding-01.jpg`    |

> Поддерживаемые форматы: `.jpg`, `.jpeg`, `.webp`, `.png`  
> Нумерация: `01`, `02`, `03` и т.д. — двузначная, с ведущим нулём.

### Аватарка (фото в разделе "Обо мне")

Положи файл явно в корень папки `photos/`:

```
photos/avatar.jpg
```

Фото должно быть квадратным (лучше всего 1:1) — вырежется автоматически до круга. Если файл не найден, аватарка просто не появится — без ошибок.

### 2. Добавь запись в `photos.json`

Открой файл `photos.json` и добавь строку в нужный массив `photos`:

```json
{
  "categories": {
    "weddings": {
      "label": "Свадьбы",
      "photos": [
        { "file": "wedding-01.jpg", "alt": "Свадьба" },
        { "file": "wedding-02.jpg", "alt": "Свадьба" }
      ]
    },
    "events": {
      "label": "Мероприятия",
      "photos": [
        { "file": "event-01.jpg", "alt": "Мероприятие" }
      ]
    },
    "lovestory": {
      "label": "Лавстори",
      "photos": [
        { "file": "lovestory-01.jpg", "alt": "Лавстори" }
      ]
    }
  }
}
```

Поле `alt` — описание для скринридеров, можно оставить общим или написать конкретное.

### 3. Закоммить и запушить

```bash
git add photos/ photos.json
git commit -m "add: photos"
git push
```

Сайт обновится сам через ~1 минуту.

---

## Как убрать фотографию

1. Удали строку `{ "file": "...", "alt": "..." }` из `photos.json`
2. Опционально: удали сам файл из папки `photos/...`
3. Закоммить и запушить

Галерея покажет ровно столько фото, сколько записей в `photos.json`. Пустых ячеек не будет.

## Локальный просмотр (до деплоя)

Открыть `index.html` двойным кликом **не сработает** — браузер заблокирует `fetch()` для локальных файлов.

**Вариант 1 — VS Code Live Server:**
1. Установи расширение [Live Server](https://marketplace.visualstudio.com/items?itemName=ritwickdey.LiveServer)
2. Нажми `Go Live` в правом нижнем углу

**Вариант 2 — Node:**
```bash
npx serve .
```

## Деплой на GitHub Pages

1. Создай репозиторий на GitHub (можно публичный или приватный на платном плане)
2. Запушь все файлы в ветку `main`
3. Перейди в **Settings → Pages**
4. Source: **Deploy from a branch** → Branch: `main` → Folder: `/ (root)`
5. Нажми Save

Сайт будет доступен по адресу:
```
https://ВАШ_ЛОГИН.github.io/Photography_portfolio_webpage/
```

## Обновление контактов или биографии

Все тексты находятся прямо в `index.html`:

- **Биография** — ищи `about-text`
- **Ссылки** — ищи `contact-telegram` и `contact-instagram`
- **Заголовок страницы (SEO)** — тег `<title>` в `<head>`

---

## Рекомендации по фото

- Горизонтальные снимки (3:2 или 16:9) выглядят лучше в сетке
- Оптимальный размер: **1400–2000px** по длинной стороне
- Вес: старайся не превышать **500–800 KB** на файл (используй [Squoosh](https://squoosh.app) или [TinyJPG](https://tinyjpg.com))
- Формат `.webp` даёт наименьший вес при хорошем качестве
